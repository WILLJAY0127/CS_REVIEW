嵌套类: 就是一个辅助类, 用于辅助其他类的实现.
嵌套类可以设置为私有,一旦设置为私有, 就只能在嵌套类的内部访问.嵌套类里的属性和方案的作用域就没作用了???

嵌套类也可以设置成静态的 ,如果他没有用到外部类的东西, 就可以设置成静态的.
静态嵌套类: 就是一个没有外部类的嵌套类, 可以直接通过嵌套类名来访问.
用静态类能省内存? 因为静态类的实例是共享的, 不会因为实例而增加内存占用.
静态是不会持有外部类的实例的, 所以不会增加内存占用.
非静态会持有外部类的实例, 所以会增加内存占用.同时,如果你想释放掉外部类的实例, 当内部类的实例持有外部类的实例时, 会引用外部类的实例, 所以会导致内存泄漏.

# 如何解决这个问题
好问题！内存泄漏的解决方法主要有几种，按推荐程度排序：

## 方法一：从根源避免——用静态嵌套类（推荐！）

```java
class Outer {
    // 把非静态内部类改成静态嵌套类
    static class Nested {
        // 不访问外部类的实例成员
    }
}
```

**原理**：静态嵌套类不持有外部类的引用，根本不会产生内存泄漏。

---

## 方法二：控制内部类对象的生命周期

确保内部类对象和外部类对象一起释放：

```java
public class Outer {
    private Inner inner;
    
    class Inner { }
    
    public Inner getInner() {
        return inner;
    }
    
    // 显式释放方法
    public void dispose() {
        inner = null;  // 断开内部类的引用
    }
}

// 使用时
Outer outer = new Outer();
Outer.Inner inner = outer.getInner();
// ... 使用完后
inner = null;   // 先释放内部类
outer.dispose(); // 再释放外部类中的引用
outer = null;   // 最后释放外部类
```

---

## 方法三：使用弱引用（WeakReference）

```java
import java.lang.ref.WeakReference;

class Outer {
    static class Nested {
        private WeakReference<Outer> outerRef;  // 弱引用
        
        Nested(Outer outer) {
            this.outerRef = new WeakReference<>(outer);
        }
        
        void doSomething() {
            Outer outer = outerRef.get();  // 获取引用
            if (outer != null) {
                // 使用外部类
            } else {
                // 外部类已被回收
            }
        }
    }
}
```

**原理**：弱引用不会阻止垃圾回收，当外部类没有其他强引用时，会被正常回收。

---

## 方法四：匿名内部类的特殊处理

```java
public class Outer {
    public Runnable createRunnable() {
        return new Runnable() {
            @Override
            public void run() {
                // 这个匿名内部类持有 Outer 的引用！
            }
        };
    }
}
```

这种情况可以改成：

```java
public class Outer {
    public Runnable createRunnable() {
        // 使用静态方法或Lambda表达式
        return () -> {
            // Lambda 不会隐式持有外部类引用（除非访问外部类成员）
        };
    }
}
```

---

## 总结

| 方法 | 适用场景 | 难度 | 效果 |
|---|---|---|---|
| 静态嵌套类 | 不需要访问外部类实例成员 | ⭐ 简单 | ✅ 根本解决 |
| 控制生命周期 | 必须使用非静态内部类 | ⭐⭐ 中等 | ✅ 需要手动管理 |
| 弱引用 | 需要访问外部类但不希望阻止回收 | ⭐⭐⭐ 较难 | ✅ 自动回收 |
| Lambda表达式 | 匿名内部类场景 | ⭐ 简单 | ✅ 减少引用 |

**最佳实践**：能使用静态嵌套类就用静态嵌套类，这是最简单、最安全的方式！

# 关于sllists里的size实现
方式一: 用递归来实时计算
方式二: 用一个变量来记录size, 每次添加一个元素, 就增加1.

这是一个问题的俩种实现方式, 但是, 俩种实现方式都有自己的优缺点.
方式一: 用递归来实时计算, size计算的时间是个大问题.同时,空间占用也高.
方式二: 用一个变量来记录size, 每次添加一个元素, 就增加1. 这种将时间分摊到每次添加元素时, 空间占用也低.
但是如果size本就调用的少,而有需要add方法极致的快,可就要权衡一下了.
