## 📋 数组的定义

**数组是存放在连续内存空间上的相同类型数据的集合。**

数组可以方便地通过下标索引的方式获取到对应的数据。

---

## ✨ 数组的核心特性

### 特性一：下标从 0 开始

```python
arr = [10, 20, 30, 40]
arr[0]  # → 10（第一个元素）
arr[3]  # → 40（第四个元素）
```

### 特性二：内存空间地址连续

这是数组最重要的特性，也是理解数组操作的关键！

```
内存布局示例：
地址:  0x100  0x104  0x108  0x10C  （假设每个元素占4字节）
值:     10     20     30     40
下标:    0      1      2      3
```

**为什么连续很重要？**

因为地址连续，CPU 可以通过简单的数学计算直接定位到任意元素：

```
地址 = 起始地址 + 下标 × 每个元素的大小
```

这就是数组**随机访问**的时间复杂度为 O(1) 的原因！

---

## ⚠️ 数组操作的特点

### 访问：O(1) — 非常高效

```python
arr = [10, 20, 30, 40]
print(arr[2])  # → 30，直接定位，O(1)
```

### 删除：O(n) — 需要移动元素

由于内存地址连续，删除某个元素后，后面的元素都需要向前移动：

```
原数组: [10, 20, 30, 40, 50]
删除下标2的元素(30):
步骤1: [10, 20, _, 40, 50]    # 覆盖
步骤2: [10, 20, 40, _, 50]    # 40前移
步骤3: [10, 20, 40, 50, _]    # 50前移
结果:   [10, 20, 40, 50]
```

**核心要点**：**数组的元素是不能删的，只能覆盖。**

### 插入：O(n) — 需要移动元素

同理，在某个位置插入元素，后面的元素都需要向后移动：

```
原数组: [10, 20, 30, 40]
在下标1插入25:
步骤1: [10, _, 20, 30, 40]    # 扩容，留出空间
步骤2: [10, _, 20, 30, _]    # 40后移
步骤3: [10, _, 20, _, 40]    # 30后移
步骤4: [10, 25, 20, 30, 40]  # 插入25
结果:   [10, 25, 20, 30, 40]
```

---

## 📊 数组操作时间复杂度总结

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| **访问** | O(1) | 直接通过下标定位 |
| **搜索** | O(n) | 需要遍历数组 |
| **插入** | O(n) | 需要移动后面的元素 |
| **删除** | O(n) | 需要移动后面的元素 |

## 🧮 二维数组的内存分布

### C++ 中的二维数组

**在 C++ 中，二维数组的内存地址是连续的！**

```cpp
int array[2][3] = {
    {0, 1, 2},
    {3, 4, 5}
};

// 内存布局（连续）：
// 地址:  0x820  0x824  0x828  0x82C  0x830  0x834
// 值:     0      1      2      3      4      5
```

### Java 中的二维数组

**在 Java 中，二维数组的内存地址是不连续的！**

```java
int[][] arr = {{1, 2, 3}, {3, 4, 5}, {6, 7, 8}};

// 内存布局（不连续）：
// 第一行: [I@7852e922  →  存储 [1, 2, 3]
// 第二行: [I@4e25154f  →  存储 [3, 4, 5]
// 第三行: [I@70dea4e   →  存储 [6, 7, 8]
```

Java 的二维数组更像是一个"数组的数组"，每一行是一个独立的一维数组，它们在内存中分散存储。

### Python 中的二维数组

Python 的列表是动态数组，二维列表同样是"列表的列表"，内存分布不连续：

```python
arr = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
# 每一行是一个独立的列表对象
```

## 💡 关键认知

### 数组的本质

数组的本质是**一段连续的内存空间**，通过下标直接计算地址实现 O(1) 的随机访问。

### 为什么数组删除/插入效率低？

因为内存地址连续，删除或插入某个元素后，必须移动后面所有元素来保持连续性。

### 数组 vs 容器

| 类型 | 底层实现 | 特点 |
|------|----------|------|
| **数组** | 连续内存 | 简单高效，大小固定 |
| **vector / ArrayList** | 数组 | 动态扩容，更灵活 |

在 C++ 中，`vector` 的底层实现是数组，严格来讲 `vector` 是容器，不是数组。

## 📝 总结

| 要点 | 说明 |
|------|------|
| **定义** | 连续内存空间上的相同类型数据集合 |
| **下标** | 从 0 开始 |
| **访问** | O(1)，直接定位 |
| **插入/删除** | O(n)，需要移动元素 |
| **元素删除** | 不能真正删除，只能覆盖 |
| **二维数组** | C++ 内存连续，Java/Python 内存不连续 |
