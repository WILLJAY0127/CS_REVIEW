# 704. 二分查找

## 📌 题目链接

[力扣题目链接](https://leetcode.cn/problems/binary-search/)

## 📋 题目描述

给定一个 n 个元素有序的（升序）整型数组 `nums` 和一个目标值 `target`，写一个函数搜索 `nums` 中的 `target`，如果目标值存在返回下标，否则返回 `-1`。

### 示例

```python
# 示例 1
输入: nums = [-1, 0, 3, 5, 9, 12], target = 9
输出: 4
解释: 9 出现在 nums 中并且下标为 4

# 示例 2
输入: nums = [-1, 0, 3, 5, 9, 12], target = 2
输出: -1
解释: 2 不存在 nums 中因此返回 -1
```

### 提示

- `nums` 中的所有元素是不重复的
- `n` 将在 `[1, 10000]` 之间
- `nums` 的每个元素都将在 `[-9999, 9999]` 之间

---

## ⚠️ 二分查找的前提条件

**这是使用二分法的核心前提，必须牢记！**

### 条件一：数组必须是有序的

如果数组无序，二分查找就无法确定目标值在哪一边，算法直接失效。

### 条件二：数组中无重复元素

如果数组中有重复元素，二分查找返回的下标可能不是唯一的。

**识别技巧**：当题目描述满足以上两个条件时，一定要想到二分法！

---

## 🎯 为什么二分法难写？

二分查找的逻辑很简单，但边界条件非常容易出错：

- 到底是 `while(left < right)` 还是 `while(left <= right)`？
- 到底是 `right = middle` 还是 `right = middle - 1`？

**核心原因**：对区间的定义没有想清楚！

---

## 🧠 循环不变量规则（核心！）

**区间的定义就是不变量。**

在二分查找的过程中，保持不变量，就是在 `while` 寻找中每一次边界的处理都要坚持根据区间的定义来操作。

**写二分法的关键**：
1. **先明确区间的定义**（左闭右闭 vs 左闭右开）
2. **然后严格按照区间定义来写代码**

两种区间定义对应两种不同的写法，下面分别讲解。

---

## 📐 区间定义一：左闭右闭 [left, right]

### 区间特点

- 左边界 `left`：包含在区间内
- 右边界 `right`：包含在区间内
- 数组下标本身就是 `[0, len(nums)-1]`，和闭区间天然匹配

### 推导过程

#### 第一步：初始化边界

```python
left = 0
right = len(nums) - 1  # 右边界是数组最后一个元素的下标
```

#### 第二步：确定循环条件

```python
while left <= right:  # 因为是闭区间，left == right 时还有一个元素需要检查
```

**为什么是 `<=`？**
- 当 `left == right` 时，区间 `[left, right]` 中还有一个元素
- 如果用 `<`，会漏掉这个元素

#### 第三步：更新边界

```python
mid = left + (right - left) // 2  # 防止溢出

if nums[mid] == target:
    return mid          # 找到！返回下标
elif nums[mid] < target:
    left = mid + 1      # 目标在右半部分，mid 已检查过，排除
else:
    right = mid - 1     # 目标在左半部分，mid 已检查过，排除
```

**为什么 `right = mid - 1`？**
- 因为 `nums[mid]` 已经不等于 `target`
- 区间是闭的，`mid` 不需要再检查，所以排除

### 完整代码

```python
def search(nums, target):
    left = 0
    right = len(nums) - 1  # 左闭右闭：右边界是数组最后一个元素
    
    while left <= right:   # 左闭右闭：区间不为空的条件
        mid = left + (right - left) // 2  # 防止溢出
        
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1  # 目标在右半部分，排除 mid
        else:
            right = mid - 1  # 目标在左半部分，排除 mid
    
    return -1  # 未找到
```

## 📐 区间定义二：左闭右开 [left, right)

### 区间特点

- 左边界 `left`：包含在区间内
- 右边界 `right`：**不包含**在区间内
- 右边界初始值是数组长度

### 推导过程

#### 第一步：初始化边界

```python
left = 0
right = len(nums)  # 右边界是数组长度（不包含）
```

#### 第二步：确定循环条件

```python
while left < right:  # 因为是开区间，left == right 时区间为空
```

**为什么是 `<`？**
- 当 `left == right` 时，区间 `[left, right)` 是空的
- 没有元素需要检查

#### 第三步：更新边界

```python
mid = left + (right - left) // 2

if nums[mid] == target:
    return mid
elif nums[mid] < target:
    left = mid + 1      # 目标在右半部分，mid 已检查过，排除
else:
    right = mid         # 目标在左半部分，右开区间，mid 不包含在内，所以不用减1
```

**为什么 `right = mid`？**
- 因为右边界是开的，`right` 指向的元素本身就不在区间内
- `nums[mid]` 已经不等于 `target`，所以把 `right` 设为 `mid` 就排除了它

### 完整代码

```python
def search(nums, target):
    left = 0
    right = len(nums)  # 左闭右开：右边界是数组长度（不包含）
    
    while left < right:  # 左闭右开：区间不为空的条件
        mid = left + (right - left) // 2
        
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1  # 目标在右半部分，排除 mid
        else:
            right = mid     # 目标在左半部分，右开区间，mid 不包含在内
    
    return -1
```

---

## 📝 两种区间定义对比

| 区间定义 | 左边界 | 右边界 | 循环条件 | 更新方式 |
|----------|--------|--------|----------|----------|
| **左闭右闭** `[left, right]` | `left = 0` | `right = len(nums) - 1` | `left <= right` | `right = mid - 1` / `left = mid + 1` |
| **左闭右开** `[left, right)` | `left = 0` | `right = len(nums)` | `left < right` | `right = mid` / `left = mid + 1` |

## 🧠 从题目到代码的思考过程

#### 第一步：分析题目特征

```
题目：给定有序数组 nums 和目标值 target，查找 target 的下标
```

- ✅ 数组有序 → 符合二分查找的前提条件
- ✅ 元素不重复 → 可以唯一确定目标位置
- ❓ 目标值可能不存在 → 需要处理未找到的情况

#### 第二步：选择区间定义

**推荐选择左闭右闭 `[left, right]`**：
- 最直观，符合日常思维习惯
- 和数组下标天然匹配

#### 第三步：推导核心逻辑

按照区间定义，逐步写出代码：
1. 初始化边界
2. 确定循环条件
3. 更新边界
4. 处理未找到的情况

#### 第四步：验证边界情况

| 边界情况 | 测试用例 | 预期结果 |
|----------|----------|----------|
| 目标在数组开头 | `nums=[1,3,5], target=1` | 返回 0 |
| 目标在数组末尾 | `nums=[1,3,5], target=5` | 返回 2 |
| 目标不存在 | `nums=[1,3,5], target=2` | 返回 -1 |
| 数组只有一个元素 | `nums=[5], target=5` | 返回 0 |
| 数组只有一个元素且不存在 | `nums=[5], target=3` | 返回 -1 |

## 🔄 递归实现

```python
def search(nums, target):
    def helper(left, right):
        if left > right:  # 区间为空
            return -1
        
        mid = left + (right - left) // 2
        
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            return helper(mid + 1, right)
        else:
            return helper(left, mid - 1)
    
    return helper(0, len(nums) - 1)
```

---

## ⚠️ 常见错误

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| `mid = (left + right) // 2` 溢出 | 当 left 和 right 都很大时，和可能超出整数范围 | 改用 `mid = left + (right - left) // 2` |
| 循环条件写成 `left < right` | 对于左闭右闭区间，会漏掉最后一个元素 | 改为 `left <= right` |
| 更新 `right = mid` 或 `left = mid` | 可能导致死循环 | 根据区间定义正确更新 |

| 维度 | 复杂度 |
|------|--------|
| **时间复杂度** | O(log n) — 每次搜索范围缩小一半 |
| **空间复杂度** | O(1) — 只使用常数级别的额外空间 |

---

## 💡 使用场景

二分查找适用于：
1. **有序数组**的查找问题
2. 需要**高效查找**（O(log n) 优于 O(n)）
3. 数据量较大时（n > 1000）优势明显

**注意**：如果数组无序，需要先排序，但排序本身需要 O(n log n)，此时不如直接遍历。

## 📝 总结

**二分查找的核心是区间定义！**

1. **明确区间定义**：左闭右闭 `[left, right]` 或左闭右开 `[left, right)`
2. **保持循环不变量**：每次边界处理都要坚持根据区间的定义来操作
3. **严格按照区间定义写代码**：循环条件、更新方式都由区间定义决定

只要掌握了这个核心思想，二分法就不会再写乱了！
