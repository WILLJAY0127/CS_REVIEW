# 704. 二分查找

## 📌 题目链接

[力扣题目链接](https://leetcode.cn/problems/binary-search/)

## 📋 题目描述

给定一个 n 个元素有序的（升序）整型数组 `nums` 和一个目标值 `target`，写一个函数搜索 `nums` 中的 `target`，如果目标值存在返回下标，否则返回 `-1`。

### 示例

```python
# 示例 1
输入: nums = [-1, 0, 3, 5, 9, 12], target = 9
输出: 4
解释: 9 出现在 nums 中并且下标为 4

# 示例 2
输入: nums = [-1, 0, 3, 5, 9, 12], target = 2
输出: -1
解释: 2 不存在 nums 中因此返回 -1
```

### 提示

- `nums` 中的所有元素是不重复的
- `n` 将在 `[1, 10000]` 之间
- `nums` 的每个元素都将在 `[-9999, 9999]` 之间

---

## 🧠 解题思路

### 核心思想

二分查找的本质是：**每次将搜索范围缩小一半**。

### 关键条件

1. **数组必须有序**（这是二分查找的前提）
2. **数组元素不重复**（否则可能返回任意一个匹配的下标）

### 从题目到代码的思考过程

#### 第一步：分析题目特征

```
题目：给定有序数组 nums 和目标值 target，查找 target 的下标
```

- ✅ 数组有序 → 符合二分查找的前提条件
- ✅ 元素不重复 → 可以唯一确定目标位置
- ❓ 目标值可能不存在 → 需要处理未找到的情况

#### 第二步：选择区间定义

**为什么选择左闭右闭 `[left, right]`？**
- 最直观，符合日常思维习惯
- 数组下标本身就是 `[0, len(nums)-1]`，和闭区间天然匹配

**区间定义决定了一切**：
- 右边界初始值：`right = len(nums) - 1`（因为闭区间包含最后一个元素）
- 循环条件：`left <= right`（闭区间，left == right 时还有元素需要检查）
- 更新方式：
  - 目标在右边：`left = mid + 1`（mid 已经检查过，排除）
  - 目标在左边：`right = mid - 1`（mid 已经检查过，排除）

#### 第三步：推导核心逻辑

```python
# 初始状态
left = 0
right = len(nums) - 1

# 循环：只要区间不为空
while left <= right:
    # 计算中间位置（避免溢出）
    mid = left + (right - left) // 2
    
    # 三种情况
    if nums[mid] == target:
        return mid          # 找到！返回下标
    elif nums[mid] < target:
        left = mid + 1      # 目标在右半部分
    else:
        right = mid - 1     # 目标在左半部分

# 循环结束还没找到，说明不存在
return -1
```

#### 第四步：验证边界情况

| 边界情况 | 测试用例 | 预期结果 |
|----------|----------|----------|
| 目标在数组开头 | `nums=[1,3,5], target=1` | 返回 0 |
| 目标在数组末尾 | `nums=[1,3,5], target=5` | 返回 2 |
| 目标不存在 | `nums=[1,3,5], target=2` | 返回 -1 |
| 数组只有一个元素 | `nums=[5], target=5` | 返回 0 |
| 数组只有一个元素且不存在 | `nums=[5], target=3` | 返回 -1 |

## 🐍 代码实现

### 方法一：左闭右闭区间（最常用）

```python
def search(nums, target):
    left = 0
    right = len(nums) - 1  # 右边界是数组最后一个元素
    
    while left <= right:  # 因为是闭区间，left == right 时还有一个元素需要检查
        mid = left + (right - left) // 2  # 防止溢出，等价于 (left + right) // 2
        
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1  # 目标在右半部分
        else:
            right = mid - 1  # 目标在左半部分
    
    return -1  # 未找到
```

### 方法二：左闭右开区间

```python
def search(nums, target):
    left = 0
    right = len(nums)  # 右边界是数组长度（不包含）
    
    while left < right:  # 因为是开区间，left == right 时区间为空
        mid = left + (right - left) // 2
        
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1  # 目标在右半部分
        else:
            right = mid  # 目标在左半部分（右开，所以不用减1）
    
    return -1
```

### 方法三：递归实现

```python
def search(nums, target):
    def helper(left, right):
        if left > right:
            return -1
        
        mid = left + (right - left) // 2
        
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            return helper(mid + 1, right)
        else:
            return helper(left, mid - 1)
    
    return helper(0, len(nums) - 1)
```

---

## ⚠️ 常见错误

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| `mid = (left + right) // 2` 溢出 | 当 left 和 right 都很大时，和可能超出整数范围 | 改用 `mid = left + (right - left) // 2` |
| 循环条件写成 `left < right` | 对于左闭右闭区间，会漏掉最后一个元素 | 改为 `left <= right` |
| 更新 `right = mid` 或 `left = mid` | 可能导致死循环 | 根据区间定义正确更新 |

## 📊 复杂度分析

| 维度 | 复杂度 |
|------|--------|
| **时间复杂度** | O(log n) — 每次搜索范围缩小一半 |
| **空间复杂度** | O(1) — 只使用常数级别的额外空间 |

---

## 💡 使用场景

二分查找适用于：
1. **有序数组**的查找问题
2. 需要**高效查找**（O(log n) 优于 O(n)）
3. 数据量较大时（n > 1000）优势明显

**注意**：如果数组无序，需要先排序，但排序本身需要 O(n log n)，此时不如直接遍历。

---

## 📝 总结

二分查找的核心是**区间定义**，确定好左闭右闭还是左闭右开，然后严格按照区间定义来写循环条件和更新逻辑，就不会出错！

| 区间定义 | 左边界 | 右边界 | 循环条件 | 更新方式 |
|----------|--------|--------|----------|----------|
| **左闭右闭** `[left, right]` | `left = 0` | `right = len(nums) - 1` | `left <= right` | `right = mid - 1` / `left = mid + 1` |
| **左闭右开** `[left, right)` | `left = 0` | `right = len(nums)` | `left < right` | `right = mid` / `left = mid + 1` |
