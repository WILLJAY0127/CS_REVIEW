一、国外公开课（学院体系，适合搭配你伯克利课程习惯）
1. 普林斯顿 Algorithms Part I / II（Coursera）
适合人群：想要正统计算机数据结构体系，和 61B 互补，Java 授课
内容：并查集、排序、堆、BST、哈希、图、最短路径、字符串算法
✅ 优点：课程严谨、配套教材《算法》经典，教会各种数据结构什么时候选用
❌ 缺点：
几乎不教 LeetCode 面试套路（滑动窗口、DP 套路、双指针模型很少讲）
偏重理论实现，缺少 “如何把应用题转化成算法模型” 训练
定位：完善底层基础，不能当做面试刷题课
2. MIT 6.006 Introduction to Algorithms
适合人群：喜欢伯克利风格、喜欢严谨复杂度分析、数学推导
✅ 优点：清晰讲解分治、动态规划、搜索等通用算法范式（Paradigm）
课程强调：算法四大范式：分治、贪心、DP、暴力搜索（这正是归纳题目的顶层框架！）
❌ 缺点：作业偏理论证明，和国内笔试风格差距大；大量数学分析
非常适合建立顶层思维：所有算法逃不出几大范式
3. Grokking the Coding Interview（Educative.io）【面试模式天花板】
重点推荐！完美契合你的需求
核心理念：面试算法一共只有 16 种解题 Pattern（模式），所有题目都是变种
滑动窗口、双指针、快慢指针、区间合并、BFS/DFS、二分、两大 DP 模型、单调栈等
课程不讲高深理论，专门训练：
读题 → 判断属于哪种模式 → 使用对应模板解题
✅ 正好解决你的恐惧：单纯做题不会总结
✅ 纯文字交互式课程，不需要蹲视频，可以反复复盘归纳
❌ 付费；国内访问不稳定
4. NeetCode 150（免费 YouTube 系列）
欧美求职主流路线，按照题型分类刷题，一题一题拆解思路，强调思考过程
免费资源，适合搭配练习
二、中文体系课程（国内后端求职首选，适配国内笔试风格）
1. 《代码随想录》programmercarl.com（首推免费路线）
最适合自学、建立完整刷题体系，完全免费
特点：严格模块化顺序：
数组 → 链表 → 哈希表 → 字符串 → 栈与队列 → 二叉树 → 回溯 → 贪心 → DP → 单调栈 → 图论
每一类先讲通用解题模板，再循序渐进刷题；强调复盘、归纳异同。
✅ 优点：
刷题顺序科学，不会东一题西一题
大量总结「这类题通用写法、易错边界」
Java/Python 代码都有，适配你
✅ 完美解决：刷题散乱、不会归纳模型
不需要买训练营，网站文字内容完全足够自学
2. 覃超《算法面试通关 40 讲》（极客时间）
前 Facebook 面试官，经典面试向专栏
核心输出四步切题法（一套标准化思考流程，拿到任何算法题都可以套用）：
Clarify 厘清题意边界
思考暴力解，分析复杂度
寻找优化思路、匹配数据结构 / 算法模型
编码、测试 case
适合建立标准化解题流程，避免拿到题目无从下手。
短板：例题数量偏少，偏向方法论，习题需要自己额外补充。
3. 左程云算法课
优点：把 DP、递归、图论讲得非常透彻，擅长挖掘题目底层等价模型；
缺点：内容量大，部分内容偏竞赛难度，入门容易压力大。
4. 王争《数据结构与算法之美》（极客时间）
偏向工程视角的数据结构，更多讲解实际业务如何取舍；面试刷题训练偏少。

四、回答你的核心焦虑：如何做到「会总结，不只单纯解题」
优质算法课程统一都会灌输这套思维，也是你学习时要主动训练的习惯：
做完每一道题强制自问 3 个问题：
这道题属于哪一大类范式？（二分 / 双指针 / 滑动窗口 / 回溯 / DP/BFS）
为什么选用这个算法？能不能用别的方式？复杂度优劣？
如果修改题干条件，解法会怎么变化？（边界、数据范围变化）
举例：
不要只写出滑动窗口代码；
要总结：滑动窗口适用场景：连续子数组、满足单调性条件；什么时候不能用滑动窗口
Grokking、代码随想录本质就是不断带你训练这套归纳习惯。

一、CS70 完整覆盖什么（为什么能当地基）
全称：Discrete Mathematics and Probability Theory
逻辑与证明体系（最重要）
命题逻辑、谓词逻辑、直接证明、反证、逆否、数学归纳法

集合、关系、图论基础

数论基础

组合计数
排列组合、双射、鸽巢原理；复杂度估算、枚举类问题理论基础
完整离散概率论（课程后半段重头戏）
样本空间、独立性、条件概率、随机变量、期望、方差、马尔可夫不等式、切比雪夫、马尔可夫链
👉 哈希冲突分析、负载均衡、随机算法、系统流量估算、概率型面试题全部依托这里
关键区别
国内普通《离散数学》大多砍掉概率论；
CS70 独有的优势：离散数学 + 概率论融合在同一门课程，偏向计算机应用，不是纯数学系教材。
