# 算法学习路线总规划

## 🎯 核心区分：两类算法课程

| 类型 | 代表课程 | 目标 | 特点 |
|------|----------|------|------|
| **学院派算法课** | MIT 6.006、普林斯顿 Algorithms | 讲解算法原理、复杂度证明、数据结构底层实现 | 建立理论根基，但不讲面试解题套路 |
| **面试导向算法课** | 代码随想录、Grokking the Coding Interview | 提炼解题范式、题型模式，训练"看到题目识别模型" | 解决刷题散乱、不会归纳的问题 |

**你的诉求**：不希望刷一题只会一题；希望拥有一套标准框架，能够归类、总结题型、提炼通用思路。

**推荐路线**：`CS61A → CS61B（夯实数据结构基础）→ 面试体系课程 + 分类刷题`

## 📚 课程推荐

### 一、国外公开课（学院体系）

| 课程 | 适合人群 | 内容 | 优点 | 缺点 |
|------|----------|------|------|------|
| **普林斯顿 Algorithms Part I/II** | 想要正统计算机数据结构体系，和 61B 互补 | 并查集、排序、堆、BST、哈希、图、最短路径、字符串算法 | 课程严谨、配套教材《算法》经典，教会各种数据结构什么时候选用 | 几乎不教 LeetCode 面试套路，偏重理论实现 |
| **MIT 6.006** | 喜欢伯克利风格、喜欢严谨复杂度分析 | 分治、动态规划、搜索等通用算法范式 | 清晰讲解四大算法范式：分治、贪心、DP、暴力搜索 | 作业偏理论证明，和国内笔试风格差距大 |
| **Grokking the Coding Interview** | 面试模式天花板，完美契合归纳需求 | 16 种解题 Pattern（滑动窗口、双指针、BFS/DFS 等） | 训练读题→判断模式→使用模板解题，纯文字交互式课程 | 付费；国内访问不稳定 |
| **NeetCode 150** | 欧美求职主流路线 | 按题型分类刷题，一题一题拆解思路 | 免费资源，强调思考过程 | - |

### 二、中文体系课程（国内后端求职首选）

| 课程 | 推荐度 | 特点 | 优点 | 缺点 |
|------|--------|------|------|------|
| **代码随想录** | ⭐⭐⭐⭐⭐ | 严格模块化顺序，完全免费 | 刷题顺序科学，大量总结通用写法和易错边界，Java/Python 代码都有 | 不需要买训练营，网站内容足够 |
| **覃超《算法面试通关 40 讲》** | ⭐⭐⭐⭐ | 前 Facebook 面试官，四步切题法 | 建立标准化解题流程：厘清题意→暴力解→优化→编码测试 | 例题数量偏少，偏向方法论 |
| **左程云算法课** | ⭐⭐⭐⭐ | DP、递归、图论讲得透彻 | 擅长挖掘题目底层等价模型 | 内容量大，部分偏竞赛难度 |
| **王争《数据结构与算法之美》** | ⭐⭐⭐ | 偏向工程视角 | 讲解实际业务如何取舍 | 面试刷题训练偏少 |

```

    ↓

    ↓
阶段3：代码随想录完整顺序系统刷题（主线）
       + MIT6.006 讲义（理解四大算法范式）
       + 覃超 40 讲（学习标准化解题思考步骤）
    ↓
阶段4：Grokking Coding Interview（强化题型模式归纳能力）
```

### ❌ 不推荐的顺序

不要一边学 61B，同时大量刷 LeetCode 零散题目。地基没建好直接刷题，非常容易陷入只会背题，不会抽象模型。

## 💡 如何做到「会总结，不只单纯解题」

做完每一道题强制自问 3 个问题：

1. **这道题属于哪一大类范式？**（二分 / 双指针 / 滑动窗口 / 回溯 / DP / BFS）
2. **为什么选用这个算法？**能不能用别的方式？复杂度优劣？
3. **如果修改题干条件，解法会怎么变化？**（边界、数据范围变化）

**举例**：不要只写出滑动窗口代码；要总结：滑动窗口适用场景：连续子数组、满足单调性条件；什么时候不能用滑动窗口。

## 🧮 CS70 作为底层地基

### CS70 完整覆盖

| 模块 | 内容 | 重要性 |
|------|------|--------|
| **逻辑与证明体系** | 命题逻辑、谓词逻辑、直接证明、反证、逆否、数学归纳法 | ⭐⭐⭐⭐⭐ 归纳 = 递归的数学语言 |
| **集合、关系、图论基础** | 图、路径、二分图、稳定匹配、欧拉路径 | ⭐⭐⭐⭐ 图算法的理论词汇来源 |
| **数论基础** | GCD、模运算、费马小定理、RSA | ⭐⭐⭐ 哈希、随机算法底层 |
| **组合计数** | 排列组合、双射、鸽巢原理 | ⭐⭐⭐ 复杂度估算基础 |
| **离散概率论** | 样本空间、独立性、条件概率、随机变量、期望、方差 | ⭐⭐⭐⭐⭐ 哈希冲突分析、随机算法基础 |

### 两条路线对比

| 顺序 | 优点 | 缺点 |
|------|------|------|
| **61A → CS70 → 61B**（70垫底方案） | 先建立证明思维，学61B时能看懂"为什么算法正确"，Big-Theta分析、递归复杂度更容易上手 | CS70几乎没有代码实现，连续学理论容易枯燥；61B工程实现缺少前置体感 |
| **61A → 61B → 70**（校内主流） | 先掌握数据结构工程实现，再用数学工具分析算法；学习体感更顺滑 | 初学排序、BST时，只能感性理解，缺少严谨正确性证明训练 |

### CS70 给算法面试带来的优势

- **递归/DP 正确性证明**：面试官追问"如何证明你的 DP 转移方程正确？"，归纳法就是 CS70 核心训练
- **概率类问题碾压优势**：蓄水池抽样、随机快排期望复杂度、哈希冲突概率估算
- **复杂度分析不再只会套公式**：能够自己推导递归算法复杂度递推式
- **图论问题理解更深**：二分图匹配、稳定匹配、路径存在性问题
- **系统设计加分项**：缓存、流量、抽样、A/B 实验量化讨论

### 自学避坑忠告

1. CS70 最大难点不是公式，是书写严谨证明；作业强制自己完整手写证明
2. 短期目标 1-2 年内求职，不建议极端顺序：61A→全程死磕 70 再碰 61B；编程手感会严重断层
3. CS70 不教任何代码刷题技巧，不要指望读完 70 就能秒解 LeetCode
