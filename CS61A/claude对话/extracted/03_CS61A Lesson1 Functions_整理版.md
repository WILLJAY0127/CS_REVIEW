## 第一层：编程语言的三个基本能力

任何强大的编程语言必须有三件事：
1. **原始表达式**：最基本的构建块，比如数字 `42`、函数 `max`
2. **组合方式**：把简单的东西组合成复杂的，比如 `add(2, mul(3, 4))`
3. **抽象方式**：给复杂的东西起个名字，以后直接用名字调用

这三件事贯穿整门课。递归、高阶函数、OOP，都是这三件事的延伸。

---

## 第二层：表达式和调用表达式

调用表达式的求值规则：先对 operator 和所有 operand 求值，再把函数应用到参数上。嵌套时从内到外，像拆洋葱：

```python
sub(pow(2, add(1, 10)), pow(2, 5))
# 先算 add(1, 10) = 11
# 再算 pow(2, 11) = 2048
# 再算 pow(2, 5) = 32
# 最后 sub(2048, 32) = 2016
```

CS61A 刻意先用函数形式 `add(a, mul(b, c))` 来教，是为了让你看清楚求值的层级结构。

---

## 第三层：名字和环境——抽象的起点

`=` 在 Python 里不是"等于"，是**绑定**：把一个名字绑定到一个值上。

```python
radius = 10
f = max       # 名字可以绑定到函数本身
f(2, 3, 4)    # 完全合法，返回 4
```

所有绑定关系被存在**环境**里——"计算机当前记得的所有名字和值"。

和 Java 的区别：Java 的 `int x = 3` 类型固定，Python 的 `x = 3` 只是一个标签，可以随时指向任何东西。

---

## 第四层：定义函数——更强大的抽象

赋值是把名字绑定到**值**，函数定义是把名字绑定到**一段逻辑**：

```python
def square(x):
    return mul(x, x)
```

函数定义做了三件事：创建函数对象，把名字 `square` 绑定到它，函数体**此时不执行**，等被调用时才执行。

---

## 第五层：环境和帧——名字需要上下文才有意义

`square(5)` 被调用的时候，Python 怎么知道 `x` 是 5？这就是为什么需要**环境**这个正式模型——它是 Python 用来回答"这个名字现在是什么意思"的机制。

**环境图**就是把这个过程画出来的工具。环境由一系列**帧**组成：
- **全局帧**：程序启动时创建，一直存在
- **本地帧**：每次函数被调用时创建，调用结束后消失

查找名字的规则：**先找本地帧，找不到再找全局帧。**

```python
x = 10

def foo():
    x = 99      # 本地帧里的 x
    return x

foo()   # 返回 99
x       # 还是 10，全局帧的 x 没动
```

---

## 第六层：纯函数 vs 非纯函数——有些东西会逃出帧

本地帧里的东西，函数结束后就消失了。但 `print("hi")` 执行完，屏幕上的字消失了吗？没有。这说明有些函数做的事不只是在本地帧里算个值，它们还改变了外部世界——这种改变叫**副作用**。

**纯函数**：只返回值，没有任何副作用。`abs(-3)` → 返回 3，什么都没动。

**非纯函数**：除了返回值，还产生副作用。`print("hi")` → 返回 `None`，但屏幕上出现了 "hi"。

```python
print(print(1), print(2))
# 输出：1, 2, None None
# 原因：先执行 print(1)（打印1，返回None），再执行 print(2)（打印2，返回None），最后外层 print 打印两个 None
```

**我的理解**（补充进笔记）：

**函数是什么？** 是把一段逻辑抽象出来，给它起名字，并且允许接受不同的输入。好处是同一段逻辑不用重复写，传不同的入参就能得到不同的结果——比如 `square(2)` 和 `square(5)` 用的是同一个逻辑，但因为入参不同，结果不同。

## 七、Week 1 P 阶段 — Practice（HW01）

### Q1: A Plus Abs B

```python
def a_plus_abs_b(a, b):
    if b < 0:
        f = _____
    else:
        f = _____
    return f(a, b)
```

**引导过程：**
- `f(a, b)` — `f` 是一个函数
- 当 `b < 0` 时，`a + abs(b)` = `a - b`，所以 `f = sub`
- 当 `b >= 0` 时，`a + abs(b)` = `a + b`，所以 `f = add`

**答案：** `f = sub`（当 b < 0），`f = add`（当 b >= 0）

### Q2: Two of Three

```python
def two_of_three(i, j, k):
    return _____
```

**引导过程：**
- 三个数的平方和：`i*i + j*j + k*k`
- 减去最大的数的平方，就等于最小两个数的平方和
- 最大的数：`max(i, j, k)`

**答案：** `i*i + j*j + k*k - max(i,j,k)*max(i,j,k)`

**Jace 的发现：** 如果要想一行，在步骤上就不能存在依赖性。这个观察很准确——步骤之间有依赖，就需要中间变量，就需要多行。但如果能把所有步骤表达成一个嵌套的表达式，就不需要中间状态，一行就够。

### Q3: Largest Factor

```python
def largest_factor(n):
    """Return the largest factor of n that is smaller than n."""
    "*** YOUR CODE HERE ***"
```

**引导过程：**
- 从大到小找，找到第一个能整除的就是最大因数
- 从 `n-1` 开始，往 `1` 找
- `n % i == 0` 表示 i 能整除 n

**答案：**
```python
i = n - 1
while i >= 1:
    if n % i == 0:
        return i
    i -= 1
```

### Q4: Hailstone


**引导过程：**
- 循环终止条件：`n == 1`

- 先打印初始值，再进入循环
- step 从 1 开始（第一个数也算一步）

**答案：**
```python
step = 1
print(n)
while n != 1:
    if n % 2:
        n = n * 3 + 1
    else:
        n = n // 2
    print(n)
    step += 1

```
