## 知识脉络(分层推进 + 层间桥梁)

```
原始值 → 组合 → 表达式 → 起名字 → 名字绑定
→ 给一段逻辑起名字(函数定义) → 名字需要上下文才有意义(环境和帧)
→ 有些东西会逃出帧(纯vs非纯/副作用)
```

### 第一层:编程语言的三个基本能力

教材开头的三件事,贯穿整门课——递归、高阶函数、OOP 都是这三件事的延伸:

1. **原始表达式**(primitive):`42`、`max`——语言里最基本的数据和内置
2. **组合方式**(combination):`add(2, mul(3, 4))`——把简单的拼成复杂的
3. **抽象方式**(abstraction):给东西**起名字**——这是抽象的起点

> **桥 1→2**:有了组合能力,下一个问题就是——这么复杂的表达式,先算哪个?

### 第二层:表达式和调用表达式

- **调用表达式**(call expression)结构:`operator(operand1, operand2)`
- **求值两步**:先对 operator 和所有 operand 求值,再把函数应用到参数
- 嵌套从内到外,**像拆洋葱**:`sub(pow(2, add(1,10)), pow(2,5)) = 2016`
- 表达式树可视化

> **桥 2→3**:表达式里的名字(如 `radius`)是什么意思?需要给东西起名字 → 抽象的起点。

### 第三层:名字和环境——抽象的起点

- `=` 不是"等于",是**绑定**
- 关键认知:名字可以绑定到**函数本身**(`f = max`;`max = 5` 会覆盖)
- 所有绑定关系存在**环境**里

> **桥 3→4**:赋值是把名字绑定到值,那能不能把名字绑定到"一段逻辑"?→ 更强大的抽象。

### 第四层:定义函数——更强大的抽象

- 函数定义 = 把名字绑定到一段逻辑
- **三件事**:创建函数对象、绑定名字、函数体此时**不执行**只存起来
- 函数签名 `square(x)`

> **关键桥 4→5**(Jace 在这一步亲自逼出来的问题):
> 学完函数定义之后,有个更根本的问题浮出来:`square(5)` 被调用时,Python 怎么知道 `x` 是 5?`x` 只是个名字,本身没有任何意义——它的意义来自"此刻我们在哪个上下文里"。这就是为什么需要**环境**这个正式模型:它是 Python 用来回答"这个名字现在是什么意思"的机制。

### 第五层:环境和帧——名字需要上下文才有意义

- **全局帧**:程序启动一直存在
- **本地帧**:每次调用临时创建,结束消失
- **查找规则**:先本地帧,再全局帧
- **环境图**(Environment Diagram):把这个过程画出来的工具

> **关键桥 5→6**(Jace 指出突兀后重写的):
> 本地帧里的东西,函数结束后就消失了,不会影响外面。但 `print("hi")` 执行完之后,屏幕上的字消失了吗?没有。这说明有些函数做的事不只是在本地帧里算个值,它们还改变了外部世界——这种改变叫**副作用**。

### 第六层:纯函数 vs 非纯函数——有些东西会逃出帧

- **纯函数**:作用都在本地帧里,只返回值
- **非纯函数**:还产生副作用(打印、写文件、改外部状态)
- 经典坑:`print(print(1), print(2))` 输出 `1` / `2` / `None None`(print 返回 None)
- **为什么区分**:纯函数更易测试、组合、不易 bug;后面函数式编程的核心

### 做作业必备:Python 语法与实用工具

- 运算符是调用表达式的简写(`2+3` = `add(2,3)`)
- 元组拆包是赋值的扩展
- 默认参数是函数签名的扩展
- 终端命令(`python3` / `-i` / `-m doctest -v`)
- Docstring 和 doctest 是 HW01 标配

## 我的理解(费曼推导链,保留原话)

> **函数是什么?** 是把一段逻辑抽象出来,给它起名字,并且允许接受不同的输入。好处是同一段逻辑不用重复写,传不同的入参就能得到不同的结果——比如 `square(2)` 和 `square(5)` 用的是同一个逻辑,但因为入参不同,结果不同。
>
> (Claude 纠正:如果只是起名字、没有参数,那这个函数永远只能做同一件事,和一个变量没什么区别。函数真正强大的地方是**命名 + 参数**的组合。)
>
> **怎么找到这个函数?** 通过命名。不管是自己写的还是 import 进来的,在当前环境里都有一个名字绑定。
>
> **函数怎么知道对哪个数字平方?** 要传入参。`def square(x)` 里的 `x` 只是占位符,调用的时候才绑定到具体的值。
>
> **这个绑定记录在哪里?** 调用函数时 Python 创建一个本地帧,把入参绑定进去。函数在这个帧里执行,结束后帧消失,返回值传回给调用者。
>
> **帧消失了,函数做的所有事也消失了吗?** 不对。`print("hi")` 执行完,帧消失了,但屏幕上那行字还在。`print` 返回的是 `None`,那行字不是通过返回值传出来的——它直接发生在帧外面。
>
> **这就是副作用**:返回值在帧里产生,沿着调用链传回去;副作用直接发生在帧外面,不受帧的生命周期限制。只有返回值的叫纯函数,还有副作用的叫非纯函数。

## Java 桥梁

- **函数 vs method**:Python 的函数和 Java 的 method 本质一样——把一段逻辑封装起来,给它一个名字,反复调用。
- **类型绑定**:Java `int x = 3` 类型固定;Python `x = 3` 只是标签,随时指向别的东西(更接近"贴便利贴")。
- **作用域 vs 环境**:Java 叫 scope,Python 叫 environment/frame——概念一样,但 CS61A 用环境图让你**看见**这个过程,而不是默默依赖编译器帮你管。
- **为什么用 Python 教**:Java 里这些都是隐式的,编译器帮你管;Python 把这些全部暴露出来让你看见。
- **副作用重新审视**:你在 Java 里写的很多方法其实是非纯的(有副作用),这门课让你重新审视这件事。
- **中缀 vs 函数形式**:Java 用 `a + b * c`,Python 也支持,但 CS61A 刻意先用函数形式 `add(a, mul(b, c))` 教,是为了让你看清楚求值的层级结构。

### Q2: two_of_three

- 三数平方和 → "i*i + j*j + k*k"
- 减什么等于最小两个的平方和 → "减去最大的数的平方"
- 最大数怎么拿 → "max(i,j,k)"
- **Jace 的啊哈**:"我靠为什么想到是这种方式"
- **通用思路**(Claude):**与其找你要的,不如找你不要的,再减掉。** "找'最小两个'很麻烦,但找'最大那个'只要一个 max 就搞定。"
- Jace 反驳:"找俩个小的很麻烦么" → Claude 承认并展示替代方案 `min(i*i+j*j, j*j+k*k, i*i+k*k)`,说明"全部减最大"扩展性更好(四五个数也能用)
- **Jace 自主洞察**:**"我发现如果要想一行,在步骤上就不能存在依赖性"** ← 不在课本里

### Q3: largest_factor

- 从哪个方向找 → "从最大找"
- 从哪里开始 → "从 n 开始" → **纠错**:题目说因数要比 n 小 → "n-1"
- 不知道怎么表达整除 → Claude 提示 `a % b == 0` → "n%i==0"
- 写出 while 框架,末尾 `return None` 永远不执行 → Claude 问为什么 → "不知道" → "n 大于 1,n%1 等于多少?"(1 永远是因数,循环必在 i=1 命中)
- Claude:"先跑过再说优化,别跳步骤" → Jace 跳过直接下一题(没真正跑)

### Q4: hailstone(纠错最多的一题)

| # | 纠错点 | 过程 |
