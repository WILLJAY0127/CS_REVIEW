# 一、先分清概念：多态函数 ≠ 策略模式，分 Python / Java 两套体系讲
## 1. 什么是多态函数（Polymorphic Function，函数式/动态语言概念，CS61A高频词）
**同一个函数，能接收不同类型参数，自动适配不同类型的行为，一套函数统一处理多种数据类型。**
分两种多态：
### （1）参数多态（泛型多态，Java 有对应：泛型）
函数不绑定固定类型，任意类型都能传入，代码通用。
Python 天然动态类型，随便传：
```python
def len(x):
    return x.__len__()

len([1,2])   # list
len("abc")   # str
len((1,2,3)) # tuple
```
`len` 就是典型**多态函数**，一套逻辑兼容序列、字符串等所有实现 `__len__` 的对象。

### （2）特设多态（重载多态，Python靠协议/魔术方法，Java靠方法重载）
同一个函数，对不同类型执行完全不同逻辑。
比如内置 `+`：
- `1 + 2` 数字加法
- `"a"+"b"` 字符串拼接
- `[1]+[2]` 列表合并
`+` 是典型多态运算符（本质多态函数）。

### 核心定义（CS61A）
多态函数：不需要修改函数本身，仅传入不同类型对象，就能自动执行对应类型逻辑；依靠对象自身实现统一接口方法（魔术方法）。

## 2. 为什么 Java 很少提“多态函数”这个词？术语体系不一样
Java 是**面向对象静态语言**，不谈「多态函数」，拆成三个对应概念：
1. **方法重载（静态多态）**：同名方法，不同参数类型，编译期匹配；
2. **方法重写 / 运行时多态（子类多态）**：父类引用指向子类，调用子类实现；
3. **泛型（参数多态）**：`<T>` 通用代码，对应Python无类型限制的多态函数。

Java 所有“多态能力”都封装在**类、接口、方法**里，不存在独立游离的函数，所以行业不说“多态函数”；
Python 有一等公民函数，能单独存在，所以会单独叫「多态函数」。

## 3. 和策略模式的巨大区别（你混淆的关键点）
### 1）多态函数：底层语言级原生能力
- 层级：语法/语言内置特性；
- 目的：统一一套函数处理多种类型；
- 依赖：对象实现同一套协议方法（如 `__len__`、`__add__`）；
- 无额外设计类，原生直接用。

示例Python多态函数：
```python
def print_item(item):
    print(item.show())

# 不同类实现show，丢进同一个函数自动适配
class Apple:
    def show(self): return "苹果"
class Cat:
    def show(self): return "小猫"

print_item(Apple())
print_item(Cat())
```
`print_item` 就是多态函数。

### 2）策略模式：面向对象**设计模式**（代码架构方案，Java大量使用）
- 层级：业务代码设计套路，不是语言特性；
- 目的：把**可变化算法**单独封装成独立策略类，动态切换算法；
- 结构：统一策略接口 + 多个实现类 + 上下文持有策略对象；
- 解决：大量if-else分支臃肿问题。

Java 策略模式最简示例：
```java
// 统一策略接口
interface PayStrategy {
    void pay();
}
// 不同策略实现
class AliPay implements PayStrategy {
    public void pay(){ System.out.println("支付宝支付"); }
}
class WechatPay implements PayStrategy {
    public void pay(){ System.out.println("微信支付"); }
}
// 上下文，动态切换策略
class PayContext {
    private PayStrategy strategy;
    public PayContext(PayStrategy s){ this.strategy = s; }
    public void executePay(){ strategy.pay(); }
}
```
调用时传入不同实现类，切换支付逻辑。

## 4. 两者联系（唯一重叠点）
策略模式**底层依靠Java运行时多态（重写多态）** 实现；
但它是一套工程化封装方案，和语言原生的“多态函数”不是一回事。

## 5. 极简总结区分
1. **多态函数**：动态语言（Python）术语，语言原生特性，一个函数兼容多种对象；Java没有这个叫法，但对应能力=接口+重写+泛型；
2. **策略模式**：跨语言通用设计模式，用来拆分多变算法、消除ifelse，Java后端高频使用；
3. 两者不能划等号：一个是语言基础特性，一个是代码分层架构设计方案。
