## 递归调用的顺序
用了一个cascade函数, 
他想表达的是所有递归的级联操作都在最后那次打印之前完成.

好，我找到了你的笔记。你说的 `cascade` 函数是 CS61A 经典例子，我帮你还原它要讲的道理。

---

### cascade 函数长这样

```python
def cascade(n):
    print(n)              # ① 递归调用之前
    if n >= 10:
        cascade(n // 10)  # ② 递归调用
        print(n)          # ③ 递归调用之后
```

调用 `cascade(123)`，输出是：
```
123    ← 外层①,递的阶段
12     ← 中层①,递的阶段
1      ← 内层①,到底了(基准情形)
12     ← 中层③,归的阶段
123    ← 外层③,归的阶段
```

---

### 你说的"所有递归的级联操作都在最后那次打印之前完成"

对！对于最外层 `cascade(123)` 来说：
- ② 那行 `cascade(12)` 是一个**完整的子任务**
- 它内部要完成 12→1→12 的整个"递+归"过程
- **全部做完之后**，才会执行 ③ 那行 `print(123)`

**这讲的道理是：Python 按顺序执行。递归调用是一个函数调用，它必须"彻底完成"（递到底 + 归回来），后面的代码才会执行。**

---

### 这正好对应你说的两阶段

把 cascade 拆开看：

```
递的阶段（从外到内）:
  外层① print(123)  →  中层① print(12)  →  内层① print(1)  到底

归的阶段（从内到外）:
  内层③ 无(基准情形没有③)  ←  中层③ print(12)  ←  外层③ print(123)
```

- **① `print(n)` 在递归调用之前** → "递"阶段执行，从外往内打
- **② `cascade(n//10)`** → 整个两阶段发生在这里
- **③ `print(n)` 在递归调用之后** → "归"阶段执行，从内往外打

---

### 这个例子到底想教什么？

**两件事：**

**1. 递归调用之前 vs 之后的代码，在截然不同的阶段执行**

```python
def cascade(n):
    print(n)           # 这行在"递"的时候执行(从外到内)
    if n >= 10:
        cascade(n//10) # 这一行是分水岭
        print(n)       # 这行在"归"的时候执行(从内到外)
```

同一个函数体里，递归调用**之前**和**之后**的代码，执行时机完全不同——一个在"递"，一个在"归"。

**2. 递归调用是一个"完整的黑盒"**

你不能跳过它。`cascade(12)` 不是"执行到一半就返回"，它是**把 12 的整个递归过程（12→1→12）全部走完**，才把控制权交回给外层。

---

### 现实类比

你让下属去统计一栋楼的人数（递归调用）。下属不是回来跟你说"我统计到一半了"，他是**把整栋楼数完**，给你一个最终数字，你才会继续做你后面的事（加上你这一层）。

**递归调用之前的代码 = 你派活之前做的事**
**递归调用 = 下属完成整个子任务**
**递归调用之后的代码 = 下属回来之后，你接着做的事**

---

### 一句话总结

> **递归调用是一个完整的子任务，它必须彻底做完（递+归），后面的代码才会执行。**

这就是你笔记里"所有递归的级联操作都在最后那次打印之前完成"的意思——不是什么神秘的事，就是 **Python 按顺序执行，函数调用必须全部完成才往下走**，递归也不例外。

##实现逆级联的函数
好，反向级联和级联正好对称：

---

### cascade（级联）回顾

```python
def cascade(n):
    print(n)
    if n >= 10:
        cascade(n // 10)
        print(n)
```

`cascade(123)` 输出：
```
123    ← 递（从大到小）
12
1
12     ← 归（从小到大）
123
```

---

### inverse_cascade（反向级联）

```python
def inverse_cascade(n):
    grow(n)
    print(n)
    shrink(n)

def grow(n):
    if n >= 10:
        grow(n // 10)
        print(n // 10)

def shrink(n):
    if n >= 10:
        print(n // 10)
        shrink(n // 10)
```

`inverse_cascade(123)` 输出：
```
1      ← 递（从小到大）
12
123
12     ← 归（从大到小）
1
```

---

### 对比

| | cascade(123) | inverse_cascade(123) |
|---|---|---|
| 递的阶段 | 123 → 12 → 1（从大到小） | 1 → 12（从小到大） |
| 到底 | 1 | 123 |
| 归的阶段 | 12 → 123（从小到大） | 12 → 1（从大到小） |
| 形状 | **大→小→大**（菱形） | **小→大→小**（倒菱形） |

---

### 关键区别在哪？

cascade 用**一个函数**，靠"递归调用前 print"和"递归调用后 print"实现两个阶段。

inverse_cascade 用**三个函数**，把"递"和"归"拆成独立的函数：
- `grow`：print 放在递归调用**之后** → "归"的时候执行 → 从小到大
- `shrink`：print 放在递归调用**之前** → "递"的时候执行 → 从大到小
- 中间的 `print(n)`：最大的那个，印在正中间

**同一个道理：print 在递归调用之前还是之后，决定了它在"递"还是"归"阶段执行。**

# 计数拆分
好，"计数拆分"——这是树形递归里最经典的一类问题：**数有多少种情况**。

---

### 计数拆分是什么？

之前你见过的递归都在**算一个值**（求和、阶乘、斐波那契）。计数拆分不一样，它在**数有多少种可能**。

核心思路：**总数 = 情况A的数目 + 情况B的数目**，每种情况又递归地数下去。

---

### 例子 1：爬楼梯

> 你要爬 `n` 级台阶，每次可以爬 1 级或 2 级。有多少种爬法？

**怎么拆？** 想第一步：
- 第一步爬 1 级 → 剩 `n-1` 级要爬，有多少种爬法？
- 第一步爬 2 级 → 剩 `n-2` 级要爬，有多少种爬法？

**两种情况互不重叠，加起来就是总爬法数。**

```python
def count_ways(n):
    if n <= 1:              # 基准:0级或1级,都只有1种
        return 1
    return count_ways(n-1) + count_ways(n-2)   # 第一步爬1级 + 第一步爬2级
```

trace `count_ways(4)`：
```
第一步爬1级 → count_ways(3) → ...
第一步爬2级 → count_ways(2) → ...
```

**你发现没有？这就是斐波那契！** `count_ways(n) = fib(n+1)`。同一套数列，不同的故事。

---

### 例子 2：凑零钱

> 你有 1 元、2 元、5 元的硬币，要凑出 `n` 元，有多少种凑法？

**怎么拆？** 想第一枚硬币用哪个：
- 用 1 元 → 剩 `n-1` 元要凑
- 用 2 元 → 剩 `n-2` 元要凑
- 用 5 元 → 剩 `n-5` 元要凑

```python
def count_coins(n):
    if n == 0:
        return 1            # 凑出0元:1种(什么都不选)
    if n < 0:
        return 0            # 不可能凑出负数
    return count_coins(n-1) + count_coins(n-2) + count_coins(n-5)
```

trace `count_coins(3)`：
```
用1元 → count_coins(2) → (用1元)count_coins(1) → (用1元)count_coins(0)=1
                                       → (用2元)count_coins(0)=1  → 但2>1,返回0
                     → (用2元)count_coins(0)=1
用2元 → count_coins(1) → (用1元)count_coins(0)=1
用5元 → count_coins(-2) = 0
总: 1+1+1+1+0 = ... (这里简化,实际要仔细去重)
```

**注意：这个简单版本有重复计数问题**（先1后2和先2后1会被当成两种），真正解决要去重。但**拆分思路**是对的：按"第一枚用什么"来分叉。

---

### 计数拆分的核心规律

```
总数 = Σ (每个选择对应的子问题数目)
```
