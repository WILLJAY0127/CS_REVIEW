## 知识脉络(分层推进 + 层间桥梁)

```
函数调用创建新帧 → 多个帧同时存在 → 名称解析需要规则 → 环境模型
→ 函数体里能做什么? → 引入语句 → 复合语句 → 条件语句/布尔上下文
→ 迭代语句/状态变化 → 函数复杂了 → 设计原则 + 测试
```

### 第一层:环境模型——名称是怎么被解析的?

- 用 `square(square(3))` 嵌套例子把"多帧并存"逼到极限
- 核心规则:当前帧找 → 父帧链上溯 → Global;名称离开环境无意义
- 不对称性:调用表达式在全局环境求值,函数体在局部帧执行

> **桥 1→2**:上一层一直在说"求值表达式"。但程序不只是求值,程序还要**执行动作**——这就引出了语句(Statement)的概念。

### 第二层:语句 vs. 表达式——程序的两种基本单元

- **表达式**:有返回值
- **语句**:没有值,描述"解释器下一步做什么"
- `def` 为何是语句:它在帧里创建名字-函数绑定,而非算出一个数
- **复合语句**(Compound Statement):`<header>:` + `<suite>`,头部控制代码块

> **桥 2→3**:有了"语句"的概念,条件语句就水到渠成了。

### 第三层:条件语句——让程序能做判断

- `if/elif/else` 头部是需判断真假的表达式 → 引出**布尔上下文**
- 假值:`0`、`None`、空字符串;其余为真值
- **短路求值**:`and`/`or` 返回操作数本身,不一定是 `True`/`False`

> **桥 3→4**:有了判断(`if`),还缺什么?**重复**。

### 第四层:迭代——让程序能重复

- `while` 规则:求值头部 → 真则执行 suite 回到第1步 → 假则结束
- 深层问题:`while` 能做的递归也能做,为何需要 `while`?→ `while` 改变**状态(state)**,靠赋值改帧中绑定推进
- Fibonacci 例:`pred, curr, k` 每轮都在变,进度存在变量值里

> **桥 4→5**:现在你有了条件和迭代,函数体变复杂了。这正是引入**函数设计原则**的时机。

### 第五层:函数设计原则(Ch. 1.4)

- 一个函数只做一件事
- DRY(Don't Repeat Yourself)
- Docstring(`help()` 可查)
- 默认参数值
- **为何此时讲**:有了 `if`/`while` 复杂度急升,为 Hog 项目打基础

> **桥 5→6**:为什么在这里引入测试?因为一旦你开始写有 `if/while` 的函数,手动心算执行结果就很难了——你需要工具来验证。

### 第六层:测试——你怎么知道函数是对的?

- `assert` 语句:失败抛错
- **Doctest**:测试写进 docstring,`python3 -m doctest file.py`
- 测试是"从外部证明函数行为符合预期"的唯一可靠方式

## 我的理解(费曼推导链,保留原话)

I 阶段 Claude 扮演不懂编程的人,逐个逼问。以下是 Jace 自己推导出的关键洞察:

### (a) 帧是名字→值的表

- 问 `x = 3` 执行时对什么做操作 → "把数字 3 赋值给了内存中的 x"
- 追问"存名字→值对应关系的表格叫什么" → "帧?"
- Claude 确认:`执行 x = 3 → 在当前的帧里,写下一条记录:x → 3`
- 关键推导:问 `3 + 4` 执行后帧变化 → **"没有变化"** ← 由此自己得出表达式/语句区别

### (b) 表达式 vs 语句的区别(由 Jace 自己说出)

> "`x = 3`:执行完之后,帧变了(多了一条记录);`3 + 4`:执行完之后,帧没变,只是算出了一个值"

收敛:**表达式**算值帧不变;**语句**改变帧或控制其他语句。

### (c) def 在帧里存什么(frozen function object)

- 问 `def square(x): return x*x` 执行后帧变化 → "建立了一个 name square 到具体的函数方法的键值对"
- 问执行 def 时 `x * x` 有没有被计算 → "没有啊"
- Claude 揭示:整个函数体被**打包进函数对象里,冷冻在那里,等待被调用**
- def 做两件事:(1) 把参数列表+函数体打包成函数对象(冷冻);(2) 当前帧建立 `square → 这个函数对象`

### (d) 函数对象存三样东西(含父帧地址——Jace 自己推出)

- Claude 用 `circle_area` 引用全局 `pi` 的例子追问:新帧里没有 `pi`,Python 去哪找?
- Jace:**"得去 global,那得记录 global 的链表地址"** ← **父帧概念的自主推导**
- 最终函数对象画面:`square → <function: 参数x, 函数体 return x*x, 父帧=Global>`

### (e) name resolution 沿父帧链上溯

由(d)推出:调用建新帧时新帧继承父帧地址,"沿着父帧链一路往上找,直到 Global"。
**名称解析规则**:当前帧找 → 找不到去父帧 → 直到 Global;"名字离开了它的环境就没有意义"。

### (f) 调用表达式求值处 vs 函数体执行处的不对称

此点 Jace 在 Consume 阶段就从视频抓到:**"一个显式的情况就是,调用表达式和被调用的函数体是在不同环境中求值的"**

### (g) 调用表达式的完整过程(Jace 自己分步说出)

问 `square(3)` 分几步:
> "先找到 square 然后找到对应的内存对象……然后建立新的帧……然后再帧中给 3 绑定到对应的 name 上 最后就是继续执行了"

追问 3 绑定到哪个 name → "square 的形式参数上"(即 x)。
追问新帧执行完还在吗 → "不在了";全局帧变了吗 → "没变" → **由此推出 `square(3)` 是表达式不是语句**。

收敛六步:找函数对象 → 建新帧标父帧 → 绑形参 → 执行函数体 → return 传值 → 新帧销毁,调用者帧无变化。

### (h) return 作为销毁当前帧的语句

- Jace:"return 是语句吧,销毁了当前帧"
- return 归第一类"直接改变帧"(极端情况:把整个帧干掉),做两件事——把值返回出去 + 销毁当前帧

### (i) REPL 自动显示非 None 返回值(Jace 自己发现)

> "俩行 go 和 hello,可是这是在终端才会这样吧,调用的话就只有 go 吧"

追问终端对返回值做了什么 → "打印"。
**REPL 自动打印表达式返回值(非 None)**;`Go` 无引号(print 副作用),`'hello'` 有引号(REPL 显示返回值)。

### (j) if/while 区分

- Jace 把 if 称为:**"控制其他语句的语句,元语句?"**(自创词)→ Claude 收编为"复合语句"
- while 与 if 区别 → **"没什么不同,只不过每次都要回来 check 一下"**
- 收敛:if 检查一次就走;while 检查→执行→回来再检查,直到条件变假

### Q3: Pick a Digit(数字隔离公式)

返回 n 从右数第 k 位数字,k=0 返回个位,越界返回 0;提示用 `//`、`%`、`pow`。

推导:
1. 从具体例 `digit(3579,2)=5` 手算:位置0=9/位置1=7/位置2=5/位置3=3
2. 位置0怎么算 → "用 3579 % 3570"(方向对但 3570 手算的,代码写不出)→ 改问 `3579 % 10` → "9"
3. 位置1(7)分几步 → "不知道" → 提示 `3579 // 10` → "357" → 追问 357 最右位 → "哦哦 357 % 10"
4. 列规律:位置0 `//1`、位置1 `//10`、位置2 `//100`
5. **纠错点**:问 `//` 后数字与 k 关系 → 先答"10 的 k+1 次方" → Claude 用位置0(`//1`=10^0)反例 → 自改 **"哦哦 10 的 k 次方"**
6. 公式:`n // pow(10, k) % 10`(Jace 问"什么是 pow 函数",Claude 答 `pow(a,b)`=`a**b`)

**边界纠错**:问 `digit(3579,10)` 用公式算多少 → 先答"3579" → 让重算 `3579 // pow(10,10)` → 改"0" → **公式自动处理越界**(0 % 10 = 0),无需额外 if。

### Q4: middle(三数中间值公式)
